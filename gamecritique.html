Game Critique 1: BOOEYS: A GHOST’S CODE
Basic Information
Game Name
BOOEYS: A GHOST’S CODE
Developer
SimCoach Games
Platform
Android & IOS
High-level Instructional Goal
Expose underrepresented and disadvantaged young adults to potential careers in technology-related fields. Encourage kids with aptitude to pursue those careers.
Brief Description
The basic rule is simple and clear. The goal for each level is to plan a path for a Booey to follow in order to help guide a lost to a portal, based on skills and different game items and scenarios appearing in different levels. There are different challenges created by different game items and environment settings in different game levels.
Victory Condition
When you succeed in guiding a lost to the position of the portal, it will get sublimation in the portal.
Reasons of Failures
However, there are plenty of situations that could make a lost die and fail that game level. For example, a lost can drown if he walks into water. A lost will also die if he walks into fire or walk into a sad Booey.
Increase Complexity
At first, the game is pretty easy and straightforward to think about the solution. It only took seconds for me to solve the problem. And afterwards, the complexity increase along with the increase of game items with more complex rules to consider. And I think the game really got interesting only when more rules and game items appear, which make the game more challenging.
Frustration
However, even before the game has become really complex, without some better tutorials right before the game level, I overrated the difficulty degree of the third level: Can’t hurt to try.

As the name said, I tried a lot of complex solution, any of which could not work. And meanwhile, the hint “can the lost walk through fire?” doesn’t change which means if I could not figure it out at first with the hint provided, I could not seek more help from the game once I am stuck. Cognitive bias is devastated, which made me trapped in my fixed thoughts about how this level should work. And after times of struggling, I gave up and searched for tutorial in Youtube. However, the solution turned out to be extremely simple… I don’t know what should I say to myself with such silly game performance.

In the whole process of playing this game, I always feel like I am not getting enough prepared and guidance before many game levels, which led me wasting lots of time exploring different failures. I always have to try for several times with failures to become clearer of the right thing I should do and such failures are not caused by my wrong strategies to get the solutions.

Achievement
After getting through each level, the player will get a little certification of different skill sets, including problem solving, logic attention to detail and so on. However, as my personal experience of this game, I am kind of confused about the rules behind such certification of different skill sets, such as which behaviors of mine during the current level mapping to the certification I got after completing the game level. From perspectives of both a player and also a starter of educational game design, I think it would be better if the connection between skill sets achieved after each game level and game behaviors and solution in the level can be more explicit and meaningful, which may enhance the learning performance and decrease players’ confusion. Because, besides my own thinking during the game, when going through players’ reviews in Amazon, I also saw some other players have pointed out that they could not see how this game could benefit to their technical skills, algorithm skills and so on.
Learning Objectives
Prior knowledge that students need
1. Basic reading skills to make sure that learners can understand the game rules, hints and other text appearing during the game. For example, there will be a hint for each game level to help you generate the solution more effectively. Although in most cases, I don’t think these hints are actually useful.
2. The basic counting skills to make sure learners can utilize the rule of the same velocity of a lost and a booey during the same time to get a solution correctly.
What they are likely to learn
The game uses a series of increasingly difficult puzzles to engage five aptitudes critical to technical careers:
  • 1) Problem Solving
  • 2) Logic
  • 3) Attention to Detail: There will be small game items which need attention from players in order to pass the current level.
  • 4) Understanding Algorithms: Players need to utilize some parallel concepts of “if-then”, “while” and so on to come up with solution. For example, if the lost can view the fire light without blocks, it will start to move forwards.
  • 5) Building Solutions.
As players progress through the game’s 36 levels, they are gradually exposed to potential careers that utilize the same skills needed to complete each level. This exposure demonstrates the link between gameplay and future jobs in technology.
Transfer
These skills can be transferred to computer science, especially to algorithm design.
Mechanics, Dynamics & Aesthetics
Mechanics
There are three roles in the game:
Different game items and scenarios in different game levels include:
  • 1. Different game items with different constrains/affect to the game roles will appear in different scenarios.
2. Environment will be set differently to create different levels of difficulty.

Players need to utilize the different game scenario setting and game items for different levels to let booey guide the lost to the portal successfully. There are basically three types of behaviors including:

  • 1. Trigger the lost to move forward
  • 2. Guide the lost to avoid being killed by fire, sad Booeys and water.
  • 3. Design the path to lead the lost to the portal.
This is basically that part that is mostly related to the learning goals. Such mechanics create problems which need corresponding skills to solve.

Dynamics
There is no time limitation for this game. The action of game players and the action of game characters are separated by an action button. First, the game status is static and game players can draw a path that they design for their booeys to follow. And after pressing the action button, game players lose their control to the game status and game characters start to act based on the path drawn by game players. No time limitation provide opportunity for game players to have enough time to analyze the situation, pay attention to the details and create a solution which can meet all the constraints provided by the scenarios.
Aesthetics
The background music, the visual design and the animation design are all very cute and make the game enjoyable.
Learning Principles
1 Scaffolding
There is a hint provided at the very beginning of every game level.
2 Goal-directed practice coupled with targeted feedback
Goal-directed practice coupled with targeted feedback enhances the quality of students’ learning. There will be goals shown explicitly at the end of every game level after game players succeed passing it. The goals are skillsets achieved by passing through this game level. And there are feedback to point out players’ pros and cons, and also encourage learners move on.
3. Organization Effects
Outlining, integrating, and synthesizing information produces better learning than rereading materials or other more passive strategies.

In this game, all game elements including the game characters, game environment settings, are integrated and need players to process them proactively. A player needs to first pay attention to all the details in the playgournd, and then consider attributions of each game element. Finally, with all the constraints and requirements in mind, the player will come up with a solution to the problem.
