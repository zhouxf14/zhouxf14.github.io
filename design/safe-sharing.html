Safe-Sharing  - A personal safe with 3 user scenarios
What is it?

This is a safe which can be shared with guests. Different spaces with different locks make the sharing possible. Facial recognition and speech recognition are used as its doublet code. Once discovering unidentified users, a spare empty space will be activated and the current user can only open that empty space, which is actually a fake space and will protect valuable items in the safe.

The idea of that "fake space" used to protect owner's possession when the safe is broken into came from a tiny magic box I played in my childhood.

I am the independent designer and developer for this project. I designed the spatial model for product construction with SketchUp. The safe was developed with Python and Arduino programming on Galileo Board and ZOC Terminal.

What problems does it solve?

Safe is always a very secret and personal space for people. But differentiate the space for it will provide opportunity to share such safe between the owner and guests when it is necessary, even can be "shared" safely with people who have bad intentions.

How does it work?

First, it will use face recognition to decide whether the user is its owner or not. If the face and speech password are all verified, the owner’s space will be open.

But if the face is recognized as a guest(a stranger), the user will be asked about the speech password prepared for the guest, which can prove that the user has been given the right of use by the owner.

If unfortunately the user isn’t the owner, but also not an authorized guest, then the safe will indeed be open but only a reserved empty space can be open. So nothing in the safe will be threatened by the unidentified user, who probably is a thief.
